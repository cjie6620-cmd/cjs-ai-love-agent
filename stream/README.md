# stream 模块说明

## 1. 模块定位

`stream` 是流式输出适配模块，负责 SSE 事件编码和流式响应构建。

它解决的是“模型流式结果如何稳定返回给前端”的问题。

## 2. 目录结构

- `sse.py`
  作用：SSE 工具模块。
  功能：格式化事件内容并构建标准流式响应。
- `tests/`
  作用：流式模块测试。
  功能：验证 SSE 接口与应用路由的行为。
- `__init__.py`
  作用：模块导出入口。
  功能：暴露流式工具能力。

## 3. 对外职责

- 提供 SSE 文本格式化能力。
- 提供标准流式响应封装。

## 4. 依赖边界

- 被 `api`、`agents` 等模块调用。
- 不承载业务逻辑，不直接做模型推理。
- 只聚焦流式协议表达与响应对象构建。

## 5. 企业规范做法

- 流式协议编码集中管理，避免各处自己拼 SSE 字符串。
- 事件命名、字段格式、结束标识要统一。
- 流式适配层只处理协议，不处理业务状态机。

## 6. 维护要求

- 新增流式事件类型时同步更新本 README。
- 调整 SSE 格式时同时检查前端兼容性。
