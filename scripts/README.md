# scripts 模块说明

## 1. 模块定位

`scripts` 是运维与研发辅助脚本目录，负责放置一次性或半自动化的工程脚本。

这些脚本不属于运行时主链路，但对验证、评测、导入和诊断很重要。

## 2. 目录结构

- `langsmith_eval.py`
  作用：LangSmith 评测脚本。
  功能：加载评测用例、执行评分并上传评测结果。

## 3. 对外职责

- 提供研发辅助能力。
- 支持离线评测、批处理和诊断任务。

## 4. 依赖边界

- 脚本可以依赖运行时模块。
- 运行时主链路不要反向依赖脚本目录。
- 脚本执行方式、输入来源和输出结果要明确。

## 5. 企业规范做法

- 脚本只做明确的工程辅助任务，不把核心业务逻辑长期沉淀在脚本里。
- 脚本命名要体现用途，例如 `*_eval.py`、`*_import.py`、`*_check.py`。
- 关键脚本要说明输入、输出和执行方式。

## 6. 维护要求

- 新增脚本时同步更新本 README。
- 对多人共用脚本，优先补参数说明和样例命令。
