# api 模块说明

## 1. 模块定位

`api` 是 HTTP 接口接入层，负责把外部请求转换成内部服务调用，并把结果按统一协议返回。

它只做接口组织、参数接收、依赖注入和响应封装，不承载复杂业务逻辑。

## 2. 目录结构

- `deps.py`
  作用：FastAPI 依赖定义。
  功能：提供容器获取、限流校验等公共依赖。
- `routers/chat.py`
  作用：聊天接口路由。
  功能：处理普通聊天与会话查询入口。
- `routers/health.py`
  作用：健康检查路由。
  功能：提供服务存活与基础状态探针。
- `routers/knowledge.py`
  作用：知识库管理路由。
  功能：提供上传、索引、重建、检索等接口。
- `routers/stream.py`
  作用：流式聊天路由。
  功能：提供 SSE 输出入口。
- `__init__.py`
  作用：API 总入口。
  功能：聚合路由注册。

## 3. 对外职责

- 暴露标准 HTTP API。
- 做请求校验、依赖注入和错误响应转换。
- 对接应用容器与业务服务。

## 4. 依赖边界

- 可以依赖：`contracts`、`core`、`agents`、`rag`、`security`、`stream`
- 不应依赖具体数据库实现细节。
- 不在路由层编排业务流程，不在路由层拼 Prompt。

## 5. 企业规范做法

- 每类接口独立路由文件，避免单文件过大。
- 路由层只调用 Service，不直接操作仓储和模型。
- 请求与响应统一使用 `contracts` 中的契约对象。
- 依赖注入统一放在 `deps.py` 或容器中，避免在路由里散落实例化逻辑。
- 所有新增接口都要有健康检查、鉴权、限流和异常处理意识。

## 6. 维护要求

- 新增路由时同步更新本 README。
- 新增接口时要标明用途、输入、输出和调用服务。
