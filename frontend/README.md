# frontend 模块说明

## 1. 模块定位

`frontend` 是前端应用模块，基于 `Vue 3 + Vite + Ant Design Vue` 构建，负责聊天工作台、知识上传与流式交互界面。

它解决的是“用户如何通过 Web 界面使用 Agent 能力”的问题。

## 2. 目录结构

- `src/api/`
  作用：前端接口层。
  功能：封装认证、聊天、SSE、知识库等后端请求。
- `src/composables/`
  作用：组合式业务状态目录。
  功能：封装聊天工作台状态、登录弹窗、访客额度、流式消息和自动重发逻辑。
- `src/components/`
  作用：通用组件目录。
  功能：沉淀可复用的 UI 组件。
- `src/layouts/`
  作用：布局层。
  功能：管理页面骨架和公共布局。
- `src/router/`
  作用：路由配置。
  功能：定义页面访问路径与页面映射关系。
- `src/styles/`
  作用：样式目录。
  功能：维护主题、基础样式和全局变量。
- `src/types/`
  作用：前端类型定义。
  功能：统一接口类型与业务对象结构。
- `src/views/`
  作用：页面视图目录。
  功能：承载工作台和业务页面。
- `App.vue`
  作用：应用根组件。
  功能：挂载整体页面结构。
- `main.ts`
  作用：前端启动入口。
  功能：初始化应用、样式、路由与组件库。
- `package.json`
  作用：前端依赖与脚本声明。
  功能：定义开发、构建、预览命令。
- `vite.config.ts`
  作用：Vite 配置文件。
  功能：定义构建、代理和工程配置。
- `public/`
  作用：静态资源目录。
  功能：存放无需打包处理的静态文件。

## 3. 对外职责

- 提供聊天输入、回复渲染和流式展示。
- 提供未登录试用、登录/注册弹窗、登录后自动继续发送。
- 提供知识管理入口。
- 提供可视化工作台体验。

## 4. 依赖边界

- 前端只通过 HTTP/SSE 调用后端，不直接依赖后端内部模块。
- 页面层调用 `api/`，不要在视图里散写请求代码。
- 前端不再传可信 `user_id`；聊天身份由后端根据 JWT 或访客 Cookie 注入。
- 样式、组件、接口、类型要分层管理。

## 5. 企业规范做法

- 页面、组件、接口、类型分目录管理，避免所有代码混在 `views/`。
- 页面层只负责交互编排，不直接处理底层请求细节。
- 所有接口类型统一沉淀到 `types/` 或 `api/` 附近。
- 认证 token 统一由 `src/api/auth.ts` 和 `src/api/http.ts` 维护，请求头不要在页面里手写。
- 样式优先走主题变量和统一风格，避免页面自己写一套设计语言。
- 新增页面要同步补充路由、类型和接口说明。

## 6. 维护要求

- 新增页面或接口目录时同步更新本 README。
- 重大交互变化时补充页面职责和接口映射说明。
