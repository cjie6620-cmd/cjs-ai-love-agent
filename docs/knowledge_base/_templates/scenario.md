# 场景案例文档模板

> 本模板用于编写真实或虚构的场景案例分析文档。每个案例文档应完整呈现情境、行为、结果和分析，为 Agent 提供可参考的决策依据。

## 适用场景

- 成功案例分析（可复制的经验）
- 失败案例分析（需要避免的教训）
- 复杂情境分析（多因素交织的场景）
- 典型问题分析（高频问题的代表性案例）

## 文档结构

### 1. 元数据 (Frontmatter)

```yaml
---
title: [案例标题]
description: [案例一句话描述]
category: [分类路径，如：06_case/成功案例]
tags: [标签1, 标签2, 标签3]
related_concepts: [相关概念]
related_docs: [相关文档]
case_type: success|failure|analysis|typical
version: 1.0.0
created_at: YYYY-MM-DD
updated_at: YYYY-MM-DD
difficulty: intermediate
target_agent: 情感社交Agent
confidence_level: high
sources: [案例来源]
applicability: [适用场景]
---
```

## 编写规范

1. **情境真实**：案例要有真实感，细节丰富
2. **分析深入**：不仅要描述，更要分析为什么
3. **经验可复制**：成功案例的经验要能复制
4. **教训有意义**：失败案例的教训要有价值
5. **对比清晰**：成功和失败的行为要明确对比

## 案例来源标注

| 来源类型 | 说明 | 标注方式 |
|----------|------|----------|
| 真实案例 | 真实发生，已脱敏 | 来源：[描述]，已脱敏 |
| 虚构案例 | 模拟典型情境 | 本案例为虚构案例 |
| 混合案例 | 综合多个案例 | 综合多个真实案例改编 |

## 注意事项

- 保护隐私：真实案例必须脱敏
- 客观分析：不要有主观偏见
- 多角度看：提供多个分析视角
- 实践导向：案例要能指导实践

## 占位符说明

本模板中的占位符采用双大括号语法，复制模板后请统一替换：

- `{{title}}`：文档标题
- `{{description}}`：一句话摘要（≤100字）
- `{{category}}`：所属分类路径
- `{{tags}}`：标签数组，遵循 `00_meta/01_标签体系.md`
- `{{author}}`：作者或团队
- `{{created_at}}` / `{{updated_at}}`：日期，格式 `YYYY-MM-DD`
- `{{body}}`：正文主体，按下方结构逐节填写
