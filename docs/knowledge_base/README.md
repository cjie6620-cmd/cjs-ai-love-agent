# 恋爱与人际关系知识库（RAG 知识库总览）

> 版本：v2.0 ｜ 定位：面向恋爱与亲密关系领域 AI Agent 的结构化中文知识库
> 适用场景：检索增强生成（RAG）、对话式 AI 咨询、情感陪伴、关系辅导、话术生成

---

## 一、知识库定位

本知识库系统整理了**心理学、传播学、社会学**三大基础学科中与亲密关系相关的理论，并结合**职场、家庭、社交、亲密关系**四大现实场景，沉淀出一套可用于 AI Agent 召回的结构化中文知识体系。

核心目标：

- **理论可追溯**：每条结论均可回溯到公认的学术理论或经典著作（如依恋理论、社会渗透理论、非暴力沟通等）。
- **场景可迁移**：通过"基础 → 场景 → 技能 → 策略 → 专题 → 案例"的层级组织，保证任意问题都能定位到最相关的分片。
- **话术可生成**：提供大量可直接改写、套用的中文话术模板，便于 LLM 做 few-shot 推理。
- **安全可控**：单列安全红线与禁忌话术章节，防止 Agent 输出越界或伤害性内容。

## 三、文档命名与组织规范

- **编号前缀**：同级文档使用 `01_`、`02_` 两位数字前缀，保证排序稳定与检索可预测。
- **中文主体**：文件名与正文使用简体中文，专有名词保留原文并括注中文（例：`Attachment Theory（依恋理论）`）。
- **一级结构**：每个文档包含「概念定义 → 适用场景 → 核心要点 → 操作步骤 → 话术示例 → 常见误区 → 参考资料」七段骨架。
- **关联链接**：通过相对路径互相引用，例如：`[依恋理论](../01_foundation/心理学基础/01_依恋理论.md)`。

## 四、RAG 召回建议

### 4.1 Chunk 切分

- **粗粒度**：按 `##` 二级标题切分，单 chunk 约 400–800 tokens，适合做主题级检索。
- **细粒度**：按 `###` 三级标题 + 话术表格切分，单 chunk 约 150–300 tokens，适合做话术级检索。

### 4.2 召回策略

| 用户意图 | 推荐召回目录 | 说明 |
|----------|--------------|------|
| 理论解释类（"什么是焦虑型依恋"） | `01_foundation/` | 优先召回概念定义与理论模型 |
| 场景应对类（"见家长怎么办"） | `02_scenario/` + `03_skill/` | 先场景后技能，做二路召回 |
| 话术生成类（"怎么回复她的冷淡消息"） | `relationship_knowledge/` + `04_strategy/` | 直接召回话术模板并改写 |
| 特殊情境类（"异地恋快分手了"） | `05_special/` + `06_case/` | 专题文档与失败案例复盘 |
| 安全拦截类（PUA/极端情绪） | `relationship_knowledge/11_安全红线与禁忌话术.md` | 优先召回并输出安全兜底 |

### 4.3 元数据标注

建议向量化时注入以下 metadata：

- `category`：对应一级目录（foundation / scenario / skill / strategy / special / case）
- `stage`：关系阶段（单身 / 破冰 / 暧昧 / 恋爱 / 稳定 / 冲突 / 分手）
- `tag`：内容标签（理论 / 技能 / 话术 / 案例 / 安全）
- `audience`：目标用户（男性 / 女性 / 通用）

## 五、质量规范

1. **可读性**：段落 ≤ 5 行，优先使用表格和列表。
2. **可操作性**：每条建议需包含「做什么 / 为什么 / 怎么做 / 注意什么」四要素。
3. **可引用性**：引用学术结论须注明来源（Bowlby, 1969 / Rosenberg, 2003 等）。
4. **无歧视**：性别、性取向、年龄、婚姻状况、身份背景保持中立。
5. **反 PUA**：杜绝操纵、打压、冷暴力、情感勒索类话术。

## 六、使用建议

- **AI Agent 调用**：优先走 RAG 检索 → 召回 Top-K 片段 → 基于片段生成回答 → 若触发安全红线则启用兜底模板。
- **人工贡献**：新增文档须套用 `_templates/` 中的对应模板，并在 `00_meta/00_知识库概览.md` 登记。
- **版本管理**：重大调整在 commit message 中标注 `docs(kb): ...`，大版本升级更新本 README 的版本号。
