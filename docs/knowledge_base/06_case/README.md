# 06_case ｜ 案例分析层

> 案例是连接理论与现实的桥梁。本目录以**成功案例 + 失败案例**的双面镜方式，从真实或合成场景中提取可迁移的原则。

---

## 一、案例编写规范

- **匿名化**：所有姓名、公司、学校均做虚构处理；真实案例须脱敏。
- **结构化**：每个案例遵循「背景 → 冲突 → 关键决策 → 结果 → 可迁移原则」五段。
- **不下判决**：呈现多方视角，避免简单归因"谁对谁错"。
- **标签化**：在文档头部打上场景标签（异地 / 姐弟 / 闪婚 / 职场 …）。

## 三、如何从案例中学习

1. **分离情境与原则**：关注案例背后的通用原则，而非"照抄剧情"。
2. **双向对比**：同一主题对照成功与失败案例，找关键差异变量。
3. **关联理论**：尝试用 `01_foundation/` 的理论解释案例结果。
4. **情境迁移**：把案例原则套用到用户当前场景，必要时再调整。

---

## 四、使用建议

- 对于"讲一个例子"的用户需求，优先召回本目录并配合 `05_special/` 的专题文档。
- 给出案例时务必强调"因人而异"，避免被理解成标准答案。
- 失败案例不是为了指责当事人，而是为了提醒读者规避同类风险。
