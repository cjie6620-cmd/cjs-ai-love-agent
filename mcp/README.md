# mcp 模块说明

## 1. 模块定位

`mcp` 是 MCP 协议适配层，负责定义 MCP 协议类型、异常和传输实现。

它解决的是“项目如何以统一方式接入 MCP 服务”的问题。

## 2. 目录结构

- `types.py`
  作用：MCP 类型定义。
  功能：定义工具描述、调用结果等协议对象。
- `errors.py`
  作用：MCP 异常定义。
  功能：统一网络异常、超时异常、协议异常语义。
- `transport/base.py`
  作用：传输抽象接口。
  功能：统一约束 MCP 传输层能力。
- `transport/streamable_http.py`
  作用：Streamable HTTP 传输实现。
  功能：负责 initialize、session 管理与 MCP 请求收发。
- `__init__.py`
  作用：模块导出入口。
  功能：聚合协议层公共对象。

## 3. 对外职责

- 提供 MCP 请求发送能力。
- 提供 MCP 协议类型和异常模型。
- 为上层工具注册体系提供可复用适配。
- 支持基于 Streamable HTTP 的正式会话模型。

## 4. 依赖边界

- 可以被 `llm.tools` 等模块复用。
- 不直接承载业务规则，不做聊天编排。
- 只处理协议、传输和错误表达。

## 5. 企业规范做法

- 协议层与业务层分离，避免 MCP 细节污染业务代码。
- 统一超时、状态码、协议格式校验逻辑。
- 所有外部传输异常都转成语义明确的模块内异常。
- 新增传输方式时，优先保持接口一致。

## 6. 维护要求

- 新增传输方式或协议字段时同步更新本 README。
- 改动协议对象时检查上游 Tool Client 兼容性。
