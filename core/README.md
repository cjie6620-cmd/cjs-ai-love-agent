# core 模块说明

## 1. 模块定位

`core` 是应用基础设施装配层，负责配置管理、容器初始化和全局依赖注册。

它是项目启动阶段的基础底座，不直接承载业务能力。

## 2. 目录结构

- `config.py`
  作用：配置统一出口。
  功能：读取环境变量，统一管理应用配置项。
- `container.py`
  作用：应用容器。
  功能：集中装配 Service、Repository、Guard、Workflow 等核心依赖。
- `event_loop.py`
  作用：Windows 兼容事件循环工厂。
  功能：为本地 `uvicorn --reload` 提供兼容的事件循环创建方式。
- `startup_probe.py`
  作用：启动依赖探活。
  功能：探测 Redis、MySQL、pgvector、MinIO、Celery、MCP 等依赖状态，并提供给 `/health`。
- `tests/`
  作用：基础装配测试。
  功能：验证容器是否正确连线。
- `__init__.py`
  作用：核心模块公共导出。
  功能：暴露常用配置与容器能力。

## 3. 对外职责

- 提供全局配置读取能力。
- 提供依赖注入与应用级装配入口。
- 提供启动阶段依赖探活结果。
- 保证各模块实例创建方式一致。

## 4. 依赖边界

- 可以依赖底层基础模块。
- 不承载业务编排，不直接提供聊天能力。
- 上层模块通过容器拿依赖，避免到处手工 new 对象。

## 5. 企业规范做法

- 配置读取统一收敛，禁止业务代码直接散读环境变量。
- 容器负责装配，不写业务逻辑。
- 初始化顺序要清晰，避免循环依赖。
- 默认值、开关项、环境差异要集中在配置层维护。

## 6. 维护要求

- 新增全局配置项时同步更新本 README 和 `.env.example`。
- 新增容器依赖时要说明由谁创建、生命周期是什么。
